# Decisiones técnicas (ADRs)

Las Architecture Decision Records (ADRs) documentan
decisiones técnicas relevantes del equipo.

No sirven para justificar el pasado,
sino para dar contexto al futuro.

Usamos ADRs cuando:

- una decisión tiene impacto transversal
- hay trade-offs relevantes
- queremos evitar repetir la misma discusión

Cada ADR sigue el template estándar del repo.
